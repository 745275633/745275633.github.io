Prolog 解释器和编译器的使用

解释器

GNU

输入

gprolog

启动 GNU Prolog 解释器，会提示

GNU Prolog 版本 (位数 bits)
Compiled 编译时间 with 编译工具
By Daniel Diaz
Copyright (C) 1999-2013 Daniel Diaz

提示符是 | ?-， 次行提示符（一行没输完的提示符，如bash是>）是 （不提示）

每个（组）解后面有问号，;输出下一个（组）解，a输出全部解

使用yes和no

swi

输入

swipl

启动 GNU Prolog 解释器，会提示

Welcome to SWI-Prolog (线程, 位数 bits, Version 版本)
Copyright (c) 1990-2015 University of Amsterdam, VU Amsterdam
SWI-Prolog comes with ABSOLUTELY NO WARRANTY. This is free software,
and you are welcome to redistribute it under certain conditions.
Please visit http://www.swi-prolog.org for details.

For help, use ?- help(Topic). or ?- apropos(Word).

提示符是 ?- ，次行提示符是|

;输出下一个（组）解，w显示完整的解（太长了会以省略号显示）

使用true和false

载入文件

在解释器中输入

consult(filename).

filename替换成文件名，可以使用相对路径或绝对路径，有除了英文字母以外的字符要加单引号，没有扩展名的会自动加上.pl

或者可以直接写成

[filename].

和 consult 一样

也可以使用命令行参数

GNU 是

--consult-file FILE

FILE 换成文件名，不过写--c FILE也可以，只要前几个字母一样就行了

swi 是

-f file          User initialisation file
-l file          Script source file
-s file          Script source file

或者直接加文件名，这三个我还没发现什么区别

编译器

GNU 使用

gplc input -o output

input 换成输入文件名， output 换成输出文件名，如果不指定输出文件名，那么是 input 的基本名

swi 使用

swipl -o output -c input

如果不指定输出文件名，那么是 a.out
